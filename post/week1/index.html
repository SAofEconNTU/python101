Jul 11, 2021

Visual studio code是什麼

Visual Studio Code（簡稱VS Code）是一個由微軟開發且開放原始碼的文字編輯器，除了強大的編寫任何語言檔案的特色（畢竟他就是個文字編輯器），他還具備了許多環境開發的功能，因此也能算是被稱為一個整合開發環境（IDE）。

VS code支援偵錯、除錯（debugger），並內建Git版本控制，也支援程式碼補全（打到一半會幫你顯示出來）。它也支援許多插件（比如說自動排版、語法搜尋等），讓用戶可以自行調整設定、改變背景顏色，算是一個非常自由且用戶生態活躍的軟體。

我私心真的非常推薦、我覺得他比其他IDE厲害的是他的渲染功能，除了能寫Python、C各種程式語言以外，還可以打Latex、markdown筆記、Html live server、jupyter notebook…，總之他就是個超級酷的軟體，希望大家之後也能愛上他哈哈。

VS code 使用環境介紹

下面這張圖我大略畫了vs code 打開後會長的樣子，大家可以先簡略看看，先玩玩看不同按鍵按下去會發生什麼事。我就依序來介紹，其實它還有很多下拉式的選單我就無法一一截圖了，就到時候week1工作坊時再跟大家說明囉

  1. 文字編輯區：這邊就是打程式、編輯文件的區域。將滑鼠移到字上方就可以開始編輯囉。這邊其實就是一個IDE的本體-文字編輯器的區域了。
  2. 檔案路徑：這邊會標示著你的檔案是在哪些資料夾底下。
  3. 程式預覽區：如果學習程式的過程中，程式越來越長，很容易可以透過這個地方選取段落並跳到比如說有錯的地方。
  4. 面板（Panel）：這邊有四個區域，可以透過該區左上角的四個區域顯示不同的面板，依次是：
    • 問題（PROBLEMS）:這邊會顯示說你的程式哪裡跑錯了、變數名稱打錯了等等，如果正在除錯可以先把這邊的文字複製貼上到Google應該就會有解答了。
    • 輸出（OUTPUT）：如果用一些python的互動介面，會出現在這裡。（我超不常用）
    • 偵錯主控台（DEBUG CONSOLE）：除錯時好用的幫手，會幫你把一些錯誤訊息列在這裡。
    • 終端機（TERMINAL）：最常用，他就是把你的terminal或cmd搬過來這裡，基本上什麼都能在這裡解決（input、print、執行檔案等等）
  5. 已開啟的編輯器（Open editor）：簡單來說就是你開啟的檔案，他會顯示在右上像網頁瀏覽器這種感覺，可以點擊轉換編輯不同的檔案。
  6. 開啟的資料夾：這邊是你開啟的資料夾，會顯示該資料夾裡面的檔案。
  7. 大綱（OUTLINE）：這邊會顯示該檔案的變數、函式等結構，還蠻清楚的。
  8. 提要欄：這邊主要會有五個圖示。
    • 檔案總管：就是現在看到的這些（已開啟的編輯器、開啟的資料夾、大綱）。
    • 搜尋與取代（SEARCH）：可以直接搜尋該檔案的文字（程式碼長的時候比較好找）
    • 原始檔控制（git）如果有用github協作一些專案，就可以使用這區來進行git的整合（這區應該很久之後才會需要用到）。
    • 執行與偵錯：這邊是vs code內建的除錯器，可以自己看看，簡單來說他會一行一行跑你的程式看看哪裡有錯。
    • 延伸插件（EXTENSION）：這個很重要，是管理所有插件的地方。可以在這邊開啟插件的設定、安裝啟用插件等功能。
  9. 這邊就是設定區可以透過齒輪更改一些基礎設定如背景、帳戶等。
  10. 這塊右上角的三個鍵，依序為跑程式，python存完檔要執行按這個就對了。還有分割畫面、調整編輯器等功能。
  11. 資訊區：最下面的這區也蠻多重要的資訊的，包括你跑程式的版本（PATH）、錯誤訊息、行數等。

快捷鍵介紹

接下來介紹幾個非常常用的快捷鍵，大家可以慢慢熟悉：

  • 存檔：command+S (mac)，control+S (windows)
  • 開啟設定：command+, (mac)，無預設快捷鍵，透過齒輪開啟 (windows)
  • 加註解：command+/ (mac)，control+/ (windows)
  • 打開主命令窗（可以輸入任何指令或設定或開啟文件）：command+shift+P或是F1 (mac)，F1 (windows)
  • 關閉視窗：command+shift+W (mac)，control+shift+W (windows)

大概非常簡略快速的介紹到這邊，希望大家能多玩玩熟悉這個有趣的編輯器（我每次發現新的酷插件都有種驚喜），如果不是用VS code也沒關係，只要能執python的編輯器都是好編輯器！
